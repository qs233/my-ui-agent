# 坑：DOMSnapshot bounds 与 CSS viewport 坐标系不一定一致

## 现象

在 headed 浏览器里开启 viewport 过滤后，缩进树比 headless 少很多节点；但关闭 viewport 过滤后，两者采集到的完整 DOM 数量一致。

一次实测页面为 `https://layui.dev/docs/2/form/`，视口设置为 `1280x720`：

- headless：full DOM `3160`，viewport filtered DOM `259`
- headed：full DOM `3160`，viewport filtered DOM 原本只有 `83`

这说明问题不在页面渲染内容是否缺失，而在 viewport 过滤阶段。

## 排查结论

headed 模式下，`DOMSnapshot.layout.bounds` 和 `Page.getLayoutMetrics().cssVisualViewport` 可能不是同一个坐标系。

同一次 headed 实测里：

```text
documentElement.getBoundingClientRect().width = 1265
Page.getLayoutMetrics().cssVisualViewport.clientWidth = 1265
DOMSnapshot html bounds width = 2530
Page.getLayoutMetrics().visualViewport.clientWidth = 2530
```

也就是说：

- `cssVisualViewport` 是 CSS pixel 坐标。
- 页面里的 `getBoundingClientRect()` 也是 CSS pixel 坐标。
- 但 headed 下的 `DOMSnapshot.layout.bounds` 是另一套坐标，实测刚好是 `2x`。
- CDP 的非 css `visualViewport/contentSize` 与 `DOMSnapshot.layout.bounds` 对齐。

如果拿 `cssVisualViewport` 去过滤 `DOMSnapshot.bounds`，就会把 CSS 首屏 `720px` 当成 snapshot 坐标里的 `720px`，实际只覆盖约半个首屏，导致大量节点被误删。

## 这是不是 Playwright bug

不宜直接归因于 Playwright。

Playwright 只是通过 CDP 取数据；这里涉及的接口都是 Chromium DevTools Protocol：

- `DOMSnapshot.captureSnapshot`
- `Page.getLayoutMetrics`

更准确地说，这是 Chromium/CDP 暴露出来的坐标系差异。`cssVisualViewport` 明确是 CSS pixel；旧字段 `visualViewport/contentSize` 更接近底层 snapshot 使用的坐标。headed 模式在某些环境里会出现 backing scale / 渲染坐标和 CSS 坐标不一致，即使 `window.devicePixelRatio` 仍然可能报 `1`，所以不能用 DPR 简单修正。

## 当前工程规则

自动 viewport 过滤必须使用和 `DOMSnapshot.layout.bounds` 对齐的 viewport。

当前实现策略：

1. `captureVisibleNodes(page, { viewportFilter: true })` 通过 `Page.getLayoutMetrics()` 读取 viewport。
2. 优先使用 `metrics.visualViewport`，因为它和 `DOMSnapshot.bounds` 对齐。
3. 如果没有 `visualViewport`，再 fallback 到 `metrics.cssVisualViewport`。
4. 如果调用方传入 `viewportFilter.margin`，自动 viewport 路径会按 `visualViewport/cssVisualViewport` 的比例缩放 margin。

显式传入 `viewportFilter.viewport` 时，调用方必须保证该 viewport 已经和 `DOMSnapshot.layout.bounds` 使用同一坐标系。不要把页面 CSS 坐标里的 `{ width: 1280, height: 720 }` 直接用于 headed snapshot 过滤，除非确认 snapshot bounds 也是 CSS 坐标。

## 调试建议

遇到 headless/headed 或不同机器输出差异时，优先打印以下几组值：

```text
window.innerWidth / innerHeight
window.visualViewport.width / height
document.documentElement.getBoundingClientRect()
Page.getLayoutMetrics().cssVisualViewport
Page.getLayoutMetrics().visualViewport
Page.getLayoutMetrics().cssContentSize
Page.getLayoutMetrics().contentSize
DOMSnapshot 中 html/body 的 layout.bounds
```

重点看：

- `DOMSnapshot html bounds width / getBoundingClientRect().width`
- `DOMSnapshot html bounds width / cssContentSize.width`
- `DOMSnapshot html bounds width / contentSize.width`

如果前两个比例不是 `1`，但第三个接近 `1`，说明 snapshot bounds 与 CDP 非 css metrics 对齐，viewport 过滤不能使用 `cssVisualViewport`。
